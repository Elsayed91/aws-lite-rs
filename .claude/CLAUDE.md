# CLAUDE.md

## Project

`aws-lite-rs` — Lightweight HTTP client for AWS APIs. Codegen-driven, SigV4 request signing.

Rust edition 2024.

## Build & Test Commands

```bash
cargo check
cargo clippy -- -D warnings
cargo test --lib
cargo fmt --check

# Codegen
cd codegen && uv run python -m codegen.cli apply          # Generate + apply all APIs
cd codegen && uv run python -m codegen.cli apply --api ec2 # Single API
cd codegen && uv run python -m codegen.cli apply --dry-run  # Preview

# Integration tests (local dev only, not CI)
cargo test --test integration -- --ignored --test-threads=1 --nocapture
```

## Architecture: Three-Layer Design

| Layer | Path | Generated? | Edit? |
|-------|------|-----------|-------|
| **Types** | `src/types/*.rs` | Yes | **NEVER** — codegen overwrites |
| **Ops** | `src/ops/*.rs` | Yes | **NEVER** — codegen overwrites |
| **Test Support** | `src/test_support/*.rs` | Yes | **NEVER** — codegen overwrites |
| **API** | `src/api/*.rs` | Scaffold once | **YES** — hand-written wrappers |

## Critical Rules

### GENERATED CODE IS IMMUTABLE — ENFORCED BY HOOK

**Files in `src/types/`, `src/ops/`, and `src/test_support/` are generated by codegen and MUST NEVER be edited by hand.**

If generated code has a bug:
1. **STOP immediately**
2. **Flag the issue** to the user
3. **Propose a manifest-level solution** (edit `codegen/manifests/*.toml`)
4. **Wait for user approval**

### Other Rules

1. **To change types or ops**: edit the TOML manifest, then run codegen
2. **NEVER overwrite existing `src/api/*.rs` files**
3. **No builders** — use `Default` derive + struct literal pattern
4. **Integration tests are for local dev only** — CI runs `cargo test --lib`

### Known Codegen Quirks

- AWS Config (Aws_json1.1) uses PascalCase JSON keys — set `rename_all = "PascalCase"` in manifest

### Authentication

AWS: `aws configure` or set `AWS_ACCESS_KEY_ID`/`AWS_SECRET_ACCESS_KEY`
